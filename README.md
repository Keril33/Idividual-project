


# Система управління рієлторським агентством

**Індивідуальний проект: Лабораторна робота № 7**

##  Опис проекту
Це комплексна інформаційна система, розроблена для автоматизації бізнес-процесів рієлторського агентства. Система призначена для управління об'єктами нерухомості, клієнтами, агентами, угодами, запитами та аналітикою.

Мета проекту — автоматизація життєвого циклу угод, від реєстрації об'єкта до закриття угоди та фінансового аналізу.

##  Основний функціонал

### 1. Робота з даними та сутності
Система базується на ключових моделях даних:
* **Property (Нерухомість):** квартира/будинок, площа, адреса, ціна.
* **Client (Клієнти):** покупець, власник, орендар.
* **Agent (Рієлтори):** ім'я, досвід.
* **Deal (Угоди):** купівля/оренда, комісія.
* **Request (Запити):** запити клієнтів з параметрами бажаного житла.

### 2. Бізнес-логіка
* **CRUD операції:** Створення, читання, оновлення та видалення даних для нерухомості та клієнтів.
* **Управління статусами:** Використання статусів (`OnSale`, `Sold`, `Rented`), зміна статусів (наприклад, заброньовано -> продано) та збереження історії змін.
* **Фінанси:** Підрахунок комісії рієлтора (відсоток від угоди) та облік загального доходу агентства.
* **Автоматизація:** Автоматичний підбір об'єктів під запит клієнта на основі фільтрів.

### 3. Пошук та Аналітика
* **Розширений пошук:** Фільтрація за типом, ціною, площею, районом (наприклад, "квартири до $50 000 у центрі").
* **Статистика:**
    * Кількість успішних угод за період.
    * Загальна сума продажів та середня вартість квадратного метра.
    * Ефективність агентів (кількість угод та прибуток).
    * Історія пошуків клієнта.

### 4. Ролі та Безпека
Реалізовано розділення функціоналу меню в залежності від ролі користувача:
* **Агент:** Веде свої угоди та об'єкти.
* **Менеджер:** Має доступ до повної статистики та звітів.
* **Клієнт:** Перегляд власних запитів та історії пошуку.

## Технічний стек
* **Мова:** C#
* **Збереження даних:** JSON (серіалізація/десеріалізація) та Entity Framework Core (SQLite/SQL Server).
* **Робота з даними:** LINQ (агрегатні функції `Average`, `Sum`, `Count`, групування).
* **Мережа:** `HttpClient` для запитів до зовнішнього сервера, `Async/await` для асинхронності.
* **Інтерфейс:** Консольне меню з валідацією даних та табличним представленням.

##  Структура даних
Дані зберігаються у наступних JSON файлах:
* `properties.json` — об'єкти нерухомості.
* `clients.json` — база клієнтів.
* **`agents.json`** — список рієлторів.
* `deals.json` — історія угод.
* `requests.json` — збережені запити клієнтів.
* `users.json` — облікові записи.
